## Turn 1

### User
Level 1 Epic 작성 요청.

핵심 요청:
- Epic 수준의 Business Goal만 작성
- 구현 코드/테스트 코드/파일 생성 금지
- User Story/Task로 세분화 금지
- 출력 형식(1~11 항목) 준수

제공 맥락:
- Epic: 불변식 기반 사고 훈련 시스템 구축
- 프로젝트: Magic Square 4x4 TDD Practice
- 성공 기준(커버리지, 계약 테스트, 상수명 사용, 하드코딩 금지, 추적성 등)

### AI
요청 형식에 맞춰 Level 1 문서를 작성함:
- Epic Title
- Business Goal
- Learning Goal
- Problem Statement
- Target Learners / Users
- Scope / Non-Scope
- Success Criteria
- Key Invariants
- Traceability Rule
- Candidate User Stories for Level 2

## Turn 2

### User
Level 2 User Journey 작성 요청.

핵심 요청:
- User Journey 수준만 작성
- 구현/테스트/파일 생성 금지
- 5개 Stage 구조 고정
  1) Problem Recognition
  2) Contract Definition
  3) Domain Separation
  4) Dual-Track TDD Progress
  5) Regression Protection
- 각 Stage별 필수 필드 포함
- Journey to User Story Mapping, Traceability Link 포함

### AI
요청 형식에 맞춰 Level 2 문서를 작성함:
- Persona, Journey Goal, Journey Overview(표)
- Stage 1~5 상세 섹션(필수 필드 모두 포함)
- Journey→User Story 매핑 표
- Epic/Journey/Invariant/미래 테스트 타깃 연결 표

## Turn 3

### User
Level 3 User Stories 작성 요청.

핵심 요청:
- User Story + Acceptance Criteria만 작성
- Task 분해 금지
- 코드/테스트/파일 생성 금지
- Boundary Story와 Domain Story 분리
- 각 Story마다 보호 Contract/Invariant 명시
- Acceptance Criteria는 테스트 가능한 문장으로 작성

지정 Story:
- Story 1 입력 검증 (Boundary)
- Story 2 빈칸 좌표 탐색 (Domain)
- Story 3 누락 숫자 탐색 (Domain)
- Story 4 마방진 검증 (Domain)
- Story 5 두 가지 조합 시도 (Domain/Solver)

### AI
요청 형식에 맞춰 Level 3 문서를 작성함:
- Story Overview 표
- Story 1~5 각각:
  - Layer
  - User Story
  - Acceptance Criteria(측정 가능한 문장)
  - Protected Contract/Invariant
  - Future RED Test Direction
- Traceability Matrix 포함
